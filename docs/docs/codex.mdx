# Codex Agent Notifications <VersionBadge version="v0.15.0" />

Wave can surface Codex CLI activity in the Agent panel the same way it already does for Claude Code and opencode. The integration uses `wsh agenthook codex ...` handlers wired into Codex hooks.

Completion, error, and question notifications come from the final Codex `Stop` hook. Questions are inferred conservatively only when the last assistant message clearly presents an explicit approval or choice prompt.

## Enable Codex hooks

Codex hooks are behind a feature flag in `~/.codex/config.toml`:

```toml
[features]
codex_hooks = true
```

Per the official Codex hooks docs, Codex discovers hook configuration from `~/.codex/hooks.json` and `<repo>/.codex/hooks.json`.

## Hook setup

Add this to `~/.codex/hooks.json`:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "wsh agenthook codex posttooluse"
          }
        ]
      }
    ],
    "UserPromptSubmit": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "wsh agenthook codex userpromptsubmit"
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "wsh agenthook codex stop"
          }
        ]
      }
    ]
  }
}
```

What each hook does:

- `PostToolUse` watches Bash results and records high-confidence command failures as intermediate state so they only surface if the turn ultimately stops in error.
- `UserPromptSubmit` clears the active notification when you respond, so stale question or error states do not stick around.
- `Stop` sends the terminal notification for the Codex turn, including question prompts when the final message asks you to choose from explicit options.

## What you will see in Wave

- `completion`: Codex finished a turn and produced a final response
- `error`: Codex stopped in an error state
- `question`: Codex is waiting for your approval or choice

Notifications are keyed to the current Wave terminal block, so clicking one jumps back to the originating block and tab.

## Notes

- Codex hook support is currently experimental upstream and disabled on Windows in the official Codex docs.
- `PostToolUse` matchers currently only see `Bash`.
- Commands run with the session `cwd`, so branch and worktree metadata are resolved automatically by `wsh agenthook`.
